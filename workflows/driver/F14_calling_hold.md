# المندوب — F14: الاتصال عند 3 كم وحالة Hold والمحاولة الثانية

> الدور: المندوب | الميزة: الاتصال عند 3 كم وحالة Hold والمحاولة الثانية | القواعد المشتركة: ../00_index_overview.md (القسم 4.8)

## الهدف
تنظيم تواصل المندوب مع العميل بأمان عند الاقتراب من موقع التسليم، ومعالجة حالة عدم الرد عبر تحويل الطلب إلى Hold ثم محاولة ثانية، وصولًا لإظهار رقم العميل كاستثناء موثّق فقط عند الضرورة القصوى.

## المتطلبات المسبقة / المدخلات
- طلب نشط بحالة «في الطريق» وحساب مندوب مفعّل.
- صلاحية الموقع لحساب المسافة بين المندوب والعميل.
- نظام الاتصال داخل التطبيق (VoIP/WebRTC) بأرقام مقنّعة.

## خطوات الـ Workflow
1. أثناء التوصيل يحسب النظام المسافة لحظيًا؛ وقبل 3 كم يكون زر الاتصال مخفيًا/Disabled مع رسالة «الاتصال يتاح عند الاقتراب».
2. عند الوصول إلى 3 كم أو أقل يظهر زر الاتصال داخل التطبيق مع تنبيه الخصوصية.
3. يتصل المندوب بالعميل عبر التطبيق دون رؤية الرقم الحقيقي.
4. إذا لم يرد العميل: يضغط المندوب «لم يرد العميل» ويكتب ملاحظة فتُسجّل محاولة فاشلة.
5. يحوّل الطلب إلى Hold (سبب + وقت) ويكمل باقي التوصيلات المسندة له.
6. عند العودة لمنطقة العميل يجري محاولة اتصال ثانية ويُرسَل إشعار جديد للعميل.
7. إذا فشلت المحاولة الثانية: يظهر رقم العميل كاستثناء بعد تحذير، مع تسجيل الحدث وإشعار الأدمن.
8. تُحفظ كل الأحداث في Timeline تفاصيل الطلب.

## الشاشات والعناصر
- **الخريطة إلى العميل:** زر الاتصال يظهر فقط عند 3 كم + تنبيه خصوصية.
- **الاتصال داخل التطبيق:** زر اتصال، حالة الاتصال، سجل المحاولة.
- **Hold — العميل غير متاح:** سبب التعليق، ملاحظات، زر تأكيد Hold.
- **محاولة ثانية:** زر اتصال، إشعار جديد، خيار إظهار الرقم بعد الفشل.
- **إظهار رقم العميل:** تحذير واضح، سبب الإظهار، الرقم، تسجيل الحدث.

## الحالات والاستثناءات
- **قبل 3 كم:** الاتصال غير متاح مع رسالة توضيحية.
- **عدم رد العميل (المحاولة الأولى):** محاولة فاشلة + تحويل Hold.
- **فشل المحاولة الثانية:** إظهار الرقم كاستثناء + إشعار الأدمن.
- **Offline أثناء الاقتراب:** يتعذّر الاتصال؛ يسجّل المندوب الحالة ويعيد المحاولة عند عودة الاتصال.

## القواعد والقيود
- الاتصال داخل التطبيق فقط، بأرقام مقنّعة ورسائل مشفّرة، وأثناء الطلب النشط فقط.
- لا يظهر رقم العميل الحقيقي إلا بعد فشل محاولتين + Hold + إشعار الأدمن (حالة استثنائية موثّقة).
- زر الاتصال مرتبط حصريًا بمسافة 3 كم أو أقل.

## التكامل مع الأدوار الأخرى
- **العميل:** يظهر له زر الاتصال عند 3 كم ويستقبل إشعارات المحاولات.
- **الأدمن:** يستقبل إشعار Hold وإشعار إظهار الرقم، ويرى Timeline الكامل.
- **المطعم:** لا دور مباشر بعد تسليم البوكس للمندوب.

## الإشعارات
- إشعار للعميل عند المحاولة الأولى والمحاولة الثانية (اقتراب المندوب).
- إشعار للأدمن عند تحويل Hold وعند إظهار الرقم كاستثناء.
- تسجيل كل حدث في سجل الطلب (Timeline).
