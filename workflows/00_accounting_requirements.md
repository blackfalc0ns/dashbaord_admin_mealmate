## 1. مقدمة عامة

MealMate منصة وسيطة بين المطاعم والعملاء، تقدم اشتراكات وجبات يومية حسب برامج غذائية وباقات متنوعة. يحتاج النظام إلى **خمس آليات حسابية** أساسية:

| # | الآلية |
|---|--------|
| 1 | تصنيف المطاعم تلقائياً (Basic / Platinum / Elite) |
| 2 | تسعير الاشتراك للعميل (تصنيف + مدة + خصومات) |
| 3 | قاعدة التنويع (اللمت) بين المطاعم |
| 4 | الإلغاء ورسومه والمبلغ المسترجع |
| 5 | إنذار مبكر لمراقبة ربحية الإعدادات |

---

## 2. هيكل البيانات الأساسية

| الكيان | الوصف |
|--------|--------|
| **البرنامج الغذائي (Program)** | Fitness، Losing Weight، Body Building، Low Carb — لكل برنامج سعرات وبروتين وكارب |
| **الباقة (Bundle)** | عدد الوجبات اليومية (باقة 1: ريوق+غداء · باقة 2: ريوق+غداء+عشاء+2 سناك · باقة 3: غداء+عشاء+سناك) |
| **المطعم** | يحدد **سعراً شهرياً** لكل باقة ضمن برنامج — الشهر = **26 يوم** |
| **التصنيف (Tier)** | Basic / Platinum / Elite — لكل مطعم ولكل باقة على حدة |
| **الاشتراك** | (برنامج + باقة + تصنيف + مدة بالأيام) |

## 3. الآلية الأولى: تصنيف المطاعم

> **تفاصيل إضافية:** [`00_restaurant_classification_algorithm.md`](00_restaurant_classification_algorithm.md)

### 3.1 تحويل السعر الشهري إلى يومي

```
السعر_اليومي = السعر_الشهري / 26
```

### 3.2 مطعم واحد في الباقة

| الشرط | التصنيف |
|-------|---------|
| السعر_اليومي < 4.5 | Basic |
| السعر_اليومي ≥ 6.0 | Elite |
| غير ذلك | Platinum |

*(حدود قابلة للضبط — بالدينار الكويتي)*

### 3.3 مطعمان أو أكثر

```
μ = مجموع_الأسعار_اليومية / عدد_المطاعم

σ = √( Σ(سعر_يومي − μ)² / (عدد_المطاعم − 1) )

الحد_السفلي = μ − 0.5 × σ
الحد_العلوي = μ + 0.5 × σ

إذا السعر_اليومي ≤ الحد_السفلي  → Basic
إذا السعر_اليومي ≥ الحد_العلوي  → Elite
غير ذلك                         → Platinum
```

### 3.4 مثال

| مطعم | شهري | يومي |
|------|------|------|
| A | 83.20 | 3.20 |
| B | 156.00 | 6.00 |

μ = 4.60 · σ = 1.98 · حد سفلي = 3.61 · حد علوي = 5.59  
→ A = Basic · B = Elite

### 3.5 ملاحظات

- إعادة حساب التصنيف **فوراً** عند إضافة مطعم أو تعديل سعر.
- التصنيف الجديد يُستخدم مباشرة في التسعير للاشتراكات **الجديدة**.

## 4. الآلية الثانية: تسعير الاشتراكات

### 4.1 مفاهيم أساسية

| المفهوم | القاعدة |
|---------|---------|
| **الوصول الهرمي** | Platinum يرى Basic+Platinum · Elite يرى الجميع |
| **متوسط التكلفة** | يُحسب من مطاعم **نفس التصنيف فقط** (لا تراكمي في السعر) |
| **تكلفة المطعم اليومية** | السعر_الشهري / 26 — **بدون** عمولة المنصة |
| **عمولة المطاعم الثابتة** | **10%** من قيمة الوجبات → تكلفة صافية للمنصة = السعر_اليومي × **0.9** |

### 4.2 خطوات الحساب

**1) متوسط التكلفة اليومية للتصنيف**

```
متوسط_التكلفة = متوسط(سعر_يومي_للمطاعم_في_نفس_التصنيف_والبرنامج_والباقة)
```

**2) نسبة عمولة المنصة R (استيفاء خطي)**

الأدمن يضبط **R_max** و **R_min** (افتراضي: 30% / 15%):

```
إذا المدة ≥ 26:
    R = R_min
وإلا:
    ميل = (R_max − R_min) / 25
    R = R_max − ميل × (المدة − 1)
```

**3) سعر اليوم المعروض**

```
سعر_اليوم_للعميل = متوسط_التكلفة × (1 + R)
```

**4) الإجمالي قبل الخصم**

```
الإجمالي_قبل_الخصم = سعر_اليوم_للعميل × المدة
السعر_الأساسي = round_up(الإجمالي_قبل_الخصم)
```

`round_up` = تقريب لأقرب عدد صحيح **أعلى** (70.1 → 71).

**5) خصم المشترك (إن وُجد كود)**

```
المبلغ_بعد_الخصم = السعر_الأساسي × (1 − d)
```

حيث **d = 10%** (إعداد `خصم_المشترك`).

**6) صافي إيراد المنصة (داخلي)**

```
صافي_الإيراد = المبلغ_بعد_الخصم × (1 − a) − (متوسط_التكلفة × 0.9 × المدة)
```

حيث **a = 10%** (`عمولة_المؤثر`).

### 4.3 مثال كامل

| المدخل | القيمة |
|--------|--------|
| التصنيف | Platinum |
| متوسط التكلفة اليومية | 4.50 د.ك |
| المدة | 12 يوم |
| R | 0.30 − 0.006×11 = **23.4%** |
| سعر اليوم | 4.50 × 1.234 = 5.553 |
| الإجمالي | 5.553 × 12 = 66.636 |
| السعر الأساسي | **67** |
| بعد خصم 10% | 60.30 |
| صافي الإيراد | 60.30×0.9 − (4.50×0.9×12) = **5.67 د.ك** |

### 4.4 تغيير الأسعار

- يُطبَّق على **الاشتراكات الجديدة** فوراً فقط.
- الاشتراك النشط **لا يتأثر** بأثر رجعي.
- **الفاتورة الشهرية** توضّح الأسعار المعدّلة + **تاريخ اشتراك العميل** الأصلي.

## 5. الآلية الثالثة: قاعدة التنويع (اللمت)

### 5.1 الهدف

منع العميل من اختيار المطعم الأغلى يومياً — توزيع عادل يحمي تكاليف المنصة.

### 5.2 حساب N (المطاعم المتاحة للاختيار)

| تصنيف الاشتراك | N |
|----------------|---|
| Basic | عدد مطاعم Basic |
| Platinum | عدد Basic + عدد Platinum |
| Elite | عدد Basic + Platinum + Elite |

*(نفس البرنامج والباقة)*

### 5.3 الحد الأقصى لتكرار نفس المطعم

```
Limit = max( round_up(N / المدة), 2 )
```

- `round_up` = تقريب لأعلى.
- **الحد الأدنى = 2** — حتى لا يُجبر العميل على تغيير المطعم كل يوم في الاشتراكات الطويلة.

### 5.4 مثال

Platinum · N = 3 Basic + 4 Platinum = **7** · المدة = **20** يوم  
→ Limit = max(⌈7/20⌉, 2) = max(1, 2) = **2**  
→ نفس المطعم **مرتين كحد أقصى** خلال 20 يوماً.

## 6. الآلية الرابعة: الإلغاء واسترداد المبلغ

### 6.1 الأيام المتبقية القابلة للاسترداد

```
الأيام_المتبقية = max( إجمالي_أيام_الاشتراك − الأيام_المستخدمة − 3, 0 )
```

الـ **3 أيام** = أيام تشغيلية تُخصم آلياً.  
إذا الناتج = 0 → **لا إلغاء ولا استرداد**.

### 6.2 قيمة اليوم

```
قيمة_اليوم = المبلغ_المدفوع / إجمالي_أيام_الاشتراك
المبلغ_المستحق = قيمة_اليوم × الأيام_المتبقية
```

### 6.3 نسبة رسوم الإلغاء F (تصاعدية)

```
إذا الأيام_المتبقية ≥ 23:
    F = 0.10   (10%)
وإذا 1 ≤ الأيام_المتبقية < 23:
    ميل_الرسوم = (0.10 − 0.03) / 22 ≈ 0.0031818
    F = 0.03 + ميل_الرسوم × (الأيام_المتبقية − 1)
```

*(3% عند يوم واحد متبقي → 10% عند 23 يوم متبقي)*

### 6.4 المسترجع

```
رسوم_الإلغاء = المبلغ_المستحق × F
المبلغ_المسترجع = المبلغ_المستحق − رسوم_الإلغاء
```

### 6.5 مثال

26 يوم · دفع 140 د.ك · استخدم 10 أيام:

| الخطوة | الحساب |
|--------|--------|
| متبقي | 26 − 10 − 3 = **13** |
| قيمة اليوم | 140/26 = 5.385 |
| مستحق | 5.385 × 13 = **70.00** |
| F | 0.03 + 0.0031818×12 = **6.82%** |
| رسوم | 70 × 0.06818 = 4.77 |
| **مسترجع** | **65.23 د.ك** |

## 7. الآلية الخامسة: الإنذار المبكر للربحية

> **أداة داخلية للإدارة** — راجع F27

لكل فئة (برنامج + باقة):

**1)** `أعلى_تكلفة = max(متوسط_Basic, متوسط_Platinum, متوسط_Elite)`

**2)** سيناريو أسوأ حالة:
- المدة = أقل مدة مسموحة (افتراضي **6 أيام**)
- التكلفة = أعلى_تكلفة
- R وخصم المشترك وعمولة المؤثر كما في التسعير

**3)** حساب الربح:

```
R_test = IF(المدة≥26, R_min, R_max − 0.006×(المدة−1))
السعر_الأساسي = round_up( أعلى_تكلفة × (1+R_test) × المدة )
بعد_الخصم = السعر_الأساسي × (1 − d)
صافي_الربح = بعد_الخصم × (1 − a) − (أعلى_تكلفة × 0.9 × المدة)
```

**4)** حد الأمان:

```
حد_الأمان = 0.1 × (أعلى_تكلفة × 0.9 × المدة)
```

**5)** الحكم:

| الشرط | الحالة |
|-------|--------|
| صافي_الربح < حد_الأمان | **تحذير** |
| غير ذلك | **آمن** |

## 8. ملخص الثوابت والإعدادات

| الإعداد | الرمز | الافتراضي | ملاحظة |
|---------|-------|-----------|--------|
| عمولة المطاعم (ثابتة) | — | **10%** | من تكلفة الوجبات |
| خصم المشترك | d | **10%** | عند كود خصم |
| عمولة المؤثر | a | **10%** | من المبلغ بعد الخصم |
| أعلى عمولة منصة | R_max | **30%** | يوم 1 — يضبطه الأدمن |
| أدنى عمولة منصة | R_min | **15%** | 26 يوم+ — يضبطه الأدمن |
| ميل العمولة | m | **0.006** | (R_max−R_min)/25 |
| أقل مدة اشتراك | — | **6 أيام** | للإنذار المبكر |
| أيام الشهر المرجعي | — | **26** | شهري → يومي |
| أيام تشغيلية للإلغاء | — | **3** | قبل حساب المسترجع |
| حد أمان الربحية | — | **10%** | من التكلفة الصافية |
