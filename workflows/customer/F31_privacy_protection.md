# العميل — F31: خصوصية بيانات العميل

> الدور: العميل | الميزة: حماية الخصوصية | القواعد المشتركة: ../00_index_overview.md (القسم 4.8)

## الهدف
طمأنة العميل وحماية بياناته: لا يرى المندوب اسمه أو رقمه (موقع جغرافي فقط)، ولا يرى المطعم هويته، والاتصال آمن داخل التطبيق بأرقام مقنّعة، مع توثيق أي استثناء.

## المتطلبات المسبقة / المدخلات
- حساب عميل ببيانات شخصية وموقع توصيل.
- صلاحيات الموقع/الإشعارات بموافقة مبرّرة من العميل.

## خطوات الـ Workflow
1. عند طلب صلاحية (موقع/إشعارات/ميكروفون) يُشرح السبب قبل نافذة النظام.
2. أثناء التوصيل يُشارَك مع المندوب **الموقع الجغرافي فقط** دون اسم أو رقم.
3. يرى المطعم عدد البوكسات ونوعها والموقع العام فقط، دون هوية العميل.
4. الاتصال بالمندوب داخل التطبيق فقط، بأرقام مقنّعة ورسائل مشفّرة (راجع F14).
5. إظهار رقم العميل يحدث كاستثناء موثّق فقط بعد فشل محاولتين + إشعار الأدمن.

## الشاشات والعناصر
- **Permission Required:** شرح سبب طلب الصلاحية قبل نافذة النظام.
- **الاتصال بالمندوب:** تنبيه الخصوصية وعدم إظهار الرقم إلا استثناءً.
- **الملف الشخصي:** إدارة البيانات والإعدادات والخصوصية.

## الحالات والاستثناءات
- **رفض الصلاحية:** توضيح أثر الرفض على التتبع/الاتصال مع بدائل.
- **إظهار الرقم:** حالة استثنائية موثّقة في Timeline الطلب + إشعار الأدمن.
- **بيانات حساسة:** تُعرض للعميل فقط ولا تُكشف للأطراف الأخرى.
- **Offline:** لا يؤثر على سياسات إخفاء الهوية المطبّقة خادميًا.

## القواعد والقيود
- المندوب يرى الموقع الجغرافي (Lat/Long) فقط، دون اسم العميل أو رقمه.
- المطعم يرى عدد البوكسات ونوعها والموقع العام ووقت التوصيل، دون هوية العميل.
- اتصال آمن داخل التطبيق فقط، أرقام مقنّعة ورسائل مشفّرة.
- تشفير البيانات الحساسة وحماية عامة (CSRF/SQLi/XSS) ونسخ احتياطي ومعايير PCI DSS للدفع.

## التكامل مع الأدوار الأخرى
- **المندوب:** تطبيقه منفصل تمامًا ويرى الموقع فقط.
- **المطعم:** يرى بيانات الطلب دون هوية العميل.
- **الأدمن:** يصله إشعار ويوثّق أي استثناء لإظهار الرقم، ويشرف على الأمان.

## الإشعارات
- إشعار/طلب صلاحية مبرّر قبل الوصول للموقع أو الإشعارات.
- إشعار للأدمن (وتوثيق) عند إظهار رقم العميل استثناءً.
- إشعارات الخصوصية ضمن أحداث الطلب عند اللزوم.
