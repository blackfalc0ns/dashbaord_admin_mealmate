# المطعم — F22: رفع ومتابعة الوثائق والتراخيص

> الدور: المطعم | الميزة: وثائق المطاعم والتنبيهات | القواعد المشتركة: ../00_index_overview.md (القسم 4)

## الهدف
يسجّل المطعم بياناته ووثائقه الرسمية (السجل التجاري، عقد التأسيس، الترخيص...) مع تواريخ إصدار وانتهاء كل وثيقة.
يتابع المطعم تنبيهات انتهاء الوثائق ليحدّثها قبل إيقاف نشاطه.

## المتطلبات المسبقة / المدخلات
- وثائق رسمية سارية المفعول.
- بيانات الشركة والمالك.

## خطوات الـ Workflow
1. تسجيل بيانات المطعم/الشركة/المالك.
2. رفع: السجل التجاري، عقد التأسيس، ترخيص الشركة، وأي وثائق مطلوبة.
3. إدخال تاريخ إصدار وانتهاء لكل وثيقة.
4. إرسال الوثائق للأدمن للاعتماد.
5. متابعة حالة كل وثيقة في اللوحة (معتمدة / مرفوضة / قرب الانتهاء).
6. رفع نسخة جديدة عند التنبيه بقرب الانتهاء.

## الحالات والاستثناءات
- تنبيه قبل انتهاء الوثيقة بشهرين لرفع نسخة جديدة.
- قبل الانتهاء بشهر: إيقاف استقبال الطلبات الجديدة وإخفاء المطعم ووجباته مؤقتًا.
- استئناف النشاط فور اعتماد الوثائق المحدّثة.

## القواعد والقيود
- لا اعتماد أولي للحساب دون اعتماد الوثائق (F01).
- تنبيه قبل الانتهاء بشهرين + إيقاف قبل الانتهاء بشهر.
- لكل وثيقة تاريخ إصدار وانتهاء يتابعهما النظام.

## التكامل مع الأدوار الأخرى
- الأدمن: يراجع/يعتمد الوثائق ويتابع التنبيهات.
- العميل: لا يرى المطعم ووجباته أثناء فترة الإيقاف.
- F01: الاعتماد الأولي للحساب يعتمد على صحة الوثائق.

## الإشعارات
- تنبيه قبل انتهاء كل وثيقة بشهرين.
- إشعار الإيقاف المؤقت قبل الانتهاء بشهر.
- إشعار باعتماد أو رفض كل وثيقة.
