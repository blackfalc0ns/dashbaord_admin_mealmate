# المطعم — F31: حدود بيانات العميل الظاهرة للمطعم

> الدور: المطعم | الميزة: حماية الخصوصية | القواعد المشتركة: ../00_index_overview.md (القسم 4)

## الهدف
ضمان خصوصية العميل بعدم كشف بياناته الشخصية للمطعم.
يرى المطعم فقط ما يلزم لتجهيز البوكسات وتسليمها، دون اسم العميل أو رقمه أو عنوانه الدقيق.

## المتطلبات المسبقة / المدخلات
- نظام طلبات يعرض بيانات محدودة فقط.
- التزام المطعم بمعايير الأمان والخصوصية.

## خطوات الـ Workflow
1. استقبال الطلب ببيانات محدودة فقط.
2. الاطلاع على: عدد البوكسات، نوعها، الموقع العام للتوصيل، وقت التوصيل المطلوب.
3. تجهيز البوكسات دون معرفة هوية العميل.
4. تسليم البوكسات للسائق المعتمد (F13).
5. أي تواصل لازم يتم عبر قنوات المنصة الآمنة فقط.

## الحالات والاستثناءات
- لا يظهر اسم العميل أو رقمه أو عنوانه الدقيق للمطعم إطلاقًا.
- أي إظهار لبيانات إضافية = حالة استثنائية موثّقة بإشعار للأدمن.
- الشكاوى تُعرض للمطعم دون كشف هوية مفصّلة للعميل.

## القواعد والقيود
- المطعم يرى فقط: عدد البوكسات، نوعها (بدون اسم العميل)، الموقع العام، وقت التوصيل.
- اتصال آمن عبر المنصة فقط، بأرقام مقنّعة ورسائل مشفّرة.
- تشفير البيانات الحساسة وحماية CSRF/SQLi/XSS ونسخ احتياطي يومي.

## التكامل مع الأدوار الأخرى
- العميل: بياناته الشخصية محمية من المطعم.
- المندوب: يرى الموقع الجغرافي فقط (Lat/Long) دون اسم أو رقم.
- الأدمن: يوثّق أي استثناء لإظهار بيانات ويستقبل إشعارًا به.

## الإشعارات
- إشعار للأدمن عند أي استثناء لكشف بيانات.
- لا تتضمن إشعارات المطعم أي بيانات شخصية للعميل.
