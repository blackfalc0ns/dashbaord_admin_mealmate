# الأدمن — F22: مراجعة واعتماد وثائق المطاعم

> الدور: الأدمن | الميزة: Restaurant Documents | القواعد المشتركة: ../00_index_overview.md (القسم 4)

## الهدف
يراجع الأدمن ويعتمد وثائق المطاعم (السجل التجاري، عقد التأسيس، الترخيص...)، ويتابع تواريخ انتهائها والتنبيهات الآلية، ويضمن إيقاف المطاعم منتهية الوثائق حماية للامتثال.

## المتطلبات المسبقة / المدخلات
- بيانات المطعم/الشركة/المالك + الوثائق المرفوعة (السجل التجاري، عقد التأسيس، ترخيص الشركة...).
- تاريخ إصدار وانتهاء كل وثيقة.
- إعدادات التنبيهات الزمنية (شهرين/شهر قبل الانتهاء).

## خطوات الـ Workflow
1. يستقبل الأدمن طلب تسجيل/تحديث وثائق المطعم مع تواريخها.
2. يراجع كل وثيقة ويتحقق من صحتها واكتمالها وصلاحية تاريخها.
3. يعتمد الوثائق (فيظهر المطعم/يستمر) أو يطلب استكمالًا/تصحيحًا.
4. يتابع التنبيه الآلي قبل انتهاء الوثيقة بشهرين لطلب نسخة جديدة.
5. يتأكد أنه قبل الانتهاء بشهر يوقف النظام استقبال الطلبات الجديدة ويخفي المطعم ووجباته مؤقتًا.
6. يعيد التفعيل بعد رفع المطعم نسخة محدّثة واعتمادها.

## الحالات والاستثناءات
- وثيقة منتهية لم تُحدّث → يبقى المطعم مخفيًا حتى الاعتماد.
- وثيقة مزوّرة/غير مطابقة → رفض وتصعيد وإيقاف الحساب.
- تحديث وثيقة أثناء وجود اشتراكات نشطة → ضمان استمرار الطلبات القائمة وفق السياسات.

## القواعد والقيود
- تنبيه قبل الانتهاء بشهرين (رفع نسخة جديدة).
- قبل الانتهاء بشهر: إيقاف الطلبات الجديدة وإخفاء المطعم ووجباته مؤقتًا حتى التحديث والاعتماد.
- لا اعتماد مطعم/استمراره بوثائق ناقصة أو منتهية.
- كل قرار اعتماد/رفض موثّق بالمستخدم والوقت.

## التكامل مع الأدوار الأخرى
- **المطعم:** يرفع الوثائق ويحدّثها قبل انتهائها.
- **التسجيل (F01):** الاعتماد الأولي للمطعم يعتمد على هذه الوثائق.
- **القوائم (F24) والتغطية (F21):** تتعطل عند إخفاء المطعم لانتهاء وثائقه.

## الإشعارات
- تنبيه للمطعم قبل الانتهاء بشهرين ثم بشهر.
- إشعار للمطعم عند الاعتماد/الرفض/طلب الاستكمال.
- تنبيه للأدمن عند اقتراب انتهاء وثائق أو إخفاء مطعم آليًا.
