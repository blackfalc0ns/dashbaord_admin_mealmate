# الأدمن — F01: التسجيل واعتماد الحسابات والدخول

> الدور: الأدمن | الميزة: Auth & Onboarding | القواعد المشتركة: ../00_index_overview.md (القسم 4)

## الهدف
يضمن الأدمن أن كل حساب يدخل المنصة موثّق ومعتمد: مراجعة طلبات المطاعم والسائقين قبل تفعيلها، وإدارة حسابات العملاء (تفعيل/تعطيل)، بحيث لا يظهر أي مطعم أو يعمل أي سائق قبل موافقته الصريحة.

## المتطلبات المسبقة / المدخلات
- لوحة تحكم ويب (Angular) بصلاحيات الأدمن داخل نطاق دولته.
- طلب تسجيل مطعم يتضمّن بيانات الشركة الرسمية + الوثائق المرفوعة (راجع F22).
- طلب تسجيل سائق عام يتضمّن البيانات الشخصية والرخصة وبيانات المركبة (راجع F23).
- قائمة العملاء المسجّلين عبر التطبيق (Email/Phone + كلمة المرور + المنطقة + الموافقة على الشروط).

## خطوات الـ Workflow
1. يفتح الأدمن قائمة «طلبات التسجيل المعلّقة» مقسّمة إلى: مطاعم، سائقون عامّون.
2. يفتح طلب المطعم: يراجع البيانات الرسمية والوثائق، ويتحقق من اكتمالها وصلاحية تواريخها.
3. يتخذ قرارًا: **اعتماد** (تفعيل الحساب وإظهاره للعملاء) أو **رفض/طلب استكمال** مع ذكر السبب.
4. يفتح طلب السائق العام: يتحقق من الرخصة ووثيقة المركبة ثم يمنح الموافقة النهائية للتفعيل.
5. يدير حسابات العملاء: بحث، تفعيل، أو تعطيل حساب عند الحاجة (إساءة استخدام/طلب العميل).
6. يتابع سجل الدخول والصلاحيات، ويعالج طلبات «نسيت كلمة المرور» الاستثنائية إن لزم.

## الحالات والاستثناءات
- وثائق ناقصة أو منتهية → يبقى الحساب «قيد المراجعة» ولا يُفعّل، مع إشعار صاحب الطلب بالسبب.
- محاولة مطعم/سائق العمل قبل الاعتماد → ممنوع نظاميًا (الحساب غير ظاهر/غير قابل للإسناد).
- تعطيل حساب عميل له اشتراك نشط → يجب معالجة الاشتراك (إلغاء/تجميد) قبل أو مع التعطيل.
- Social Login للعميل مدعوم لاحقًا ولا يغيّر منطق الاعتماد للمطاعم والسائقين.

## القواعد والقيود
- لا يُفعّل حساب مطعم ولا يظهر للعملاء إلا بعد موافقة الأدمن.
- لا يُفعّل السائق العام إلا بموافقة الأدمن النهائية؛ والنظام يمنع إسناد طلب لسائق غير معتمد (راجع F23).
- الأدمن (Country Admin) يرى ويعتمد فقط حسابات نطاق دولته (راجع F28).
- كل قرار اعتماد/رفض/تعطيل يُسجّل مع المستخدم والوقت في سجل التدقيق.

## التكامل مع الأدوار الأخرى
- **المطعم:** لا تظهر قوائمه ولا يستقبل طلبات قبل اعتماد حسابه ووثائقه.
- **المندوب:** لا يستلم أي طلب قبل موافقة الأدمن النهائية على بياناته.
- **العميل:** يتأثر بتفعيل/تعطيل حسابه ويعتمد على وجود مطاعم معتمدة ليشترك.

## الإشعارات
- إشعار للمطعم/السائق عند الاعتماد أو الرفض (مع السبب) أو طلب استكمال الوثائق.
- إشعار للعميل عند تفعيل/تعطيل حسابه.
- إشعار للأدمن عند ورود أي طلب تسجيل جديد يحتاج مراجعة.
